# Architecture

The technical decisions that shape your systems for years. These prompts help you think through, document, communicate, and evaluate architectural choices — from system design through to API contracts and observability strategies.

## When to Use These

- **New system:** System design document, API contract design, data model design
- **Scaling challenge:** Scalability assessment, caching strategy, database selection
- **Cross-team alignment:** Architecture review prep, technical vision document
- **Operational maturity:** Observability strategy, SLO definition, dependency mapping
- **Evaluation:** Technology radar, vendor technical evaluation
