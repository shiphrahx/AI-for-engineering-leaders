# Team Values Workshop

Design a workshop that produces a short set of real working principles — ones that actually guide how the team decides, gives feedback, and resolves disagreement — not a list of platitudes.

## Inputs to gather

Gather these before drafting. If any are missing, ask in a single batched question — never invent the team, the reason, or the constraints. Mark anything genuinely unavailable as **Unknown**.

- **Team** — name and headcount
- **Duration** — how long the session is
- **Format** — in-person, remote, or hybrid
- **Why now** — new team forming, values feel stale, post-reorg, etc.

## Steps

1. Write **Pre-work** — 1-2 reflection questions sent ahead (e.g. a moment you were proud of how something was handled; a friction point and the value that would have helped). Keep it light; tell them it feeds the workshop.
2. Build a timed **Agenda** table — segments with time allocations and facilitation notes, sized to fit the given duration.
3. Detail the **Core Exercise** — the specific activity that generates candidate values step by step (e.g. storytelling into themes, then silent sticky-note generation, clustering, dot-voting).
4. Define the **Synthesis Method** — how to go from a wall of ideas to 4-6 usable values (clustering, naming, prioritising).
5. Give the **Output Template** for each value — not just a word, but a name, a one-sentence definition, "this means we..." behaviours, and "this doesn't mean..." to head off a common misread. Include a worked example.
6. Write **Follow-through** — what to do in the week after so the values don't die in a doc (polish within days, reference them in the next real decision, revisit in 3 months).
7. Adapt to context: for **existing values that aren't working**, run the pre-work as "what value do we claim but don't live?". For a **new team with no shared history**, replace storytelling with a "best team you've been on — what made it good?" exercise. For **in-person**, swap the digital board for physical Post-its (tactile clustering is more energising). For a **60-minute slot**, cut the drafting step — leave with candidate clusters and draft offline.
8. Assemble the output in the format below.

## Output format

```
**Team Values Workshop — [Team]**
*[Duration] | [Format] | [Tool]*

---

**Pre-work (sent [N] days before)**
[1-2 reflection questions]

---

**Agenda**
| Time | Segment | Notes |
|------|---------|-------|
| 0:00 | [segment] ([N] min) | [facilitation note] |

---

**Core exercise: [name]**
[Step-by-step instructions]

---

**Output template for each value**
```
Value name (1-3 words)
Definition (1 sentence — what it means to us)
This means we... (2-3 concrete behaviours)
This doesn't mean... (1 common misread to head off)
```
[Worked example]

---

**Follow-through**
- [Post-workshop actions to keep the values alive]
```

## Boundaries

- Never invent the team, headcount, reason, or constraints. Mark unknowns as **Unknown** and ask.
- Produce a facilitation plan, not the values themselves — the values must emerge from the team, not be prescribed by the facilitator or the model.
- Avoid platitudes and generic corporate values; the template's "this means we..." and "this doesn't mean..." exist to force specificity.
- Keep the process inclusive — design exercises (silent generation, dot-voting) so quieter voices count equally, not just the loudest in the room.

## Chaining

- No natural successor — this is usually a terminal artifact. If the workshop surfaced deeper team-health issues, offer **team-health-survey-analysis** to investigate them.
