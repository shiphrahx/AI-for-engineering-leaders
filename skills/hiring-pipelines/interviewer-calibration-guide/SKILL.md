# Interviewer Calibration Guide

Design a hands-on calibration session that aligns interviewers on standards, reduces bias, and shrinks rating variance — practical practice with real examples, not a lecture on bias.

## Inputs to gather

Gather these before designing. If any are missing, ask for them in a single batched question — never invent the panel size or the specific calibration problems. Mark anything genuinely unavailable as **Unknown** in the output.

- **Interview stage to calibrate** — e.g. system design for Senior Backend
- **Current issues** — the specific calibration problems (rating variance, tech bias, new interviewers don't know the bar)
- **Interviewer pool** — how many interviewers, experienced vs. new
- **Materials available** — recorded interviews, sample responses, or role-play capacity

## Steps

1. Design the **session format**: duration, participants (experienced + new together), frequency (e.g. quarterly plus an onboarding session per new interviewer), and materials needed.
2. Build the **calibration exercise**: have everyone independently rate the same sample (recorded with consent, or role-played), scoring named dimensions (requirement gathering, architecture quality, trade-off articulation, handling pushback) plus an overall rating; reveal all ratings simultaneously; discuss discrepancies. Repeat across samples at different quality levels (clearly strong, borderline, weak).
3. Write the **rubric alignment**: a table defining each rating level (Strong No Hire / No Hire / Hire / Strong Hire) with concrete, observable signals for this stage. Include the key calibration point that "Hire" means demonstrated level-appropriate thinking, not a perfect answer — "we're hiring engineers, not evaluating blueprints."
4. Cover the **common biases** with a counter for each: technology bias (evaluate the reasoning, not the tool choice), similarity bias ("is this valid, or just not my approach?"), halo/horn effect (rate each dimension before the overall), confidence bias (confident delivery ≠ competent design; draw out quieter candidates).
5. Design the **practice debrief**: each interviewer shares rating + top observation without interruption, disagreements are discussed with specific evidence ("I rated Hire because they addressed X"), and the group practices reaching a decision.
6. Lay out **ongoing calibration**: a shadow program for new interviewers (shadow 2, be shadowed for 2, then solo), recurring sessions with fresh samples, hiring-manager follow-up on miscalibrated ratings, and tracking interviewer rating distributions to find consistent outliers.
7. Adapt to context: with **no recorded interviews**, role-play a borderline candidate; run **separate calibrations per level** since the bar differs fundamentally (a great L3 ≠ a great L5); handle **resistance from experienced interviewers** by framing it as sharing expertise, not correcting mistakes; **measure calibration** by tracking inter-rater agreement over time.
8. Assemble the output in the format below.

## Boundaries

- Never invent the panel size or the team's calibration problems — mark unknowns as **Unknown** and ask.
- Never use real candidate recordings without consent, and never reveal candidate identities in calibration materials.
- Never write rubric levels as adjectives ("good", "strong") — each level must list observable, stage-specific signals.
- Never frame calibration as correcting individuals publicly; outlier follow-up is a private 1:1, and resistance is met by inviting expertise.

## Chaining

- After calibration, offer **interview-debrief-facilitation** to run the next decision meeting with the aligned standards.
- If the variance is being measured via funnel data, **pipeline-analytics-review** can surface stage-level rating disparities.
