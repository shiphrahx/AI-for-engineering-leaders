# Phone Screen Script

Build a 30-minute screen that efficiently assesses fit AND sells the role — strong candidates are evaluating you too — and ends with a clear advance/decline gut check.

## Inputs to gather

Gather these before writing. If any are missing, ask for them in a single batched question — never invent the role level, the technical focus, or selling points. Mark anything genuinely unavailable as **Unknown** in the output.

- **Role and level** — title and seniority (e.g. L5 Senior)
- **Key technical areas** — what to probe for this role
- **Team selling points** — what makes the role attractive to strong candidates
- **Candidate context** — applied vs. passive/sourced, if known (changes the sell/evaluate balance)

## Steps

1. Write the **opening (3 min)**: a warm intro and an explicit agenda for the 30 minutes so the candidate knows what to expect.
2. Write the **their-story segment (7 min)**: an open prompt about career arc and what they want next, plus a follow-up on their most interesting recent problem. State what to listen for (motivation, self-awareness, clear communication).
3. Build **technical calibration (10 min)**: 3-4 lightweight questions tied to the named technical areas, instruction to pick 2-3 based on the candidate's background, and for each a parenthetical "(Looking for: ...)" naming the signal. Favor real-experience questions over textbook ones.
4. Write **role-fit (5 min)**: questions that surface mutual fit honestly (ownership level, team size, greenfield appetite) — framed so the candidate can self-select out.
5. Write the **their-questions slot (5 min)** with a reminder to answer honestly; authenticity is itself a selling point.
6. Add a **post-call evaluation checklist** of yes/no signals ending in the advance/decline gut check.
7. Adapt to context: for a **junior role**, drop technical depth and probe learning ability, enthusiasm, and a project they're proud of; for a **passive candidate**, flip the ratio toward selling (~10 min) and lighter evaluation; for **volume hiring**, standardize the questions exactly for consistency; include **red flags to watch** (can't explain own work, badmouths employer at length, asks nothing, textbook answers with no real examples).
8. Assemble the output in the format below.

## Output format

```
**Phone Screen: [Role]**

**Opening (3 min)**
"[Intro + agenda for the call]"

**Their Story (7 min)**
- "[Open question about career arc and what's next]"
- Listen for: [signals]
- Follow-up: "[most interesting recent problem]"

**Technical Calibration (10 min)**
Pick 2-3 based on their background:
- *[Area]:* "[question]" (Looking for: [signal])

**Role Fit (5 min)**
- "[mutual-fit question]"

**Their Questions (5 min)**
"[invite questions]" — answer honestly; say so if you don't know.

**Evaluation Criteria**
- [ ] [signal]
- [ ] Would I want them in our interview loop? (gut check)
```

## Boundaries

- Never invent the role's technical focus or selling points — mark unknowns as **Unknown** and ask.
- Never script questions about protected characteristics, family status, or anything not role-relevant.
- Never let the script run past 30 minutes; cut depth before cutting the candidate's question time.
- Never write technical questions without stating the signal each one probes — an unscored question wastes the slot.

## Chaining

- After a candidate passes the screen, offer **take-home-exercise-design** to assess hands-on skill before the full loop.
- Once the loop completes, **interview-debrief-facilitation** runs the decision meeting.
